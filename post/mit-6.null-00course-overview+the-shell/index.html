MIT 6.null-00 Course Overview and the Shell

Notes and Exercises Solutions

Yuzhen Li
Feb 17, 2023 6 min read missing-semester, shell

In traditional coursework, CS students are taught Data Structures, Algorithms, Operating Systems, Machine Learning, etc. However, there is no specific course on the tools that must be used to put this knowledge into practice. For example, once you get into Linux, you will definitely use the Shell, and if you want to track changes to source code, you can't get around git. Most students know about these tools through their classmates or teachers or by reading some blogs online. The result of this is that knowledge of the tools is very shallow and scattered.

MIT 6.null The Missing Semester of Your CS Education designed to help computer science students fill in gaps in their education that may have been missed in traditional coursework and build a systematic understanding of CS pratical tools. The course is taught by experienced instructors and includes video lectures, readings, assignments, and programming projects. And the course materials are available online, and students can work through the material at their own speed.

Personally, most of the content of the course has already been used in my work so I've had some practice, but I'm still hoping to review and develop systematic knowledge through this open course.

Overall, MIT 6.null The Missing Semester of Your CS Education is an excellent resource for computer science students who are looking to fill in gaps in their education and develop practical skills that will help them in their careers. Whether you are a current student or a professional looking to expand your knowledge, this course is well worth exploring. With its emphasis on practical applications and flexible, self-paced learning, it offers a unique and valuable learning experience for anyone interested in computer science.

NOTES

Core of Shells: allow you to run programs, give them input, and inspect their output in a semi-structured way.

The shell parses the command by splitting it by whitespace, and then runs the program indicated by the first word, supplying each subsequent word as an argument that the program can access.

ls command: To see what lives in a given directory.

Some other handy programs to know about at this point are mv (to rename/move a file), cp (to copy a file), and mkdir (to make a new directory).

man command: It takes as an argument the name of a program, and shows you its manual page.

In the shell, programs have two primary “streams” associated with them: their input stream and their output stream.

< file and > file: Let you rewire the input and output streams of a program to a file respectively.

| operator: lets you “chain” programs such that the output of one is the input of another.

sudo command: It lets you “do” something “as su” (short for “super user”, or “root”).

EXERCISES

  1. For this course, you need to be using a Unix shell like Bash or ZSH. If you are on Linux or macOS, you don’t have to do anything special. If you are on Windows, you need to make sure you are not running cmd.exe or PowerShell; you can use Windows Subsystem for Linux or a Linux virtual machine to use Unix-style command-line tools. To make sure you’re running an appropriate shell, you can try the command echo $SHELL. If it says something like /bin/bash or /usr/bin/zsh, that means you’re running the right program.
[yuzhencode@VM-16-4-centos ~]$ echo $SHELL

  1. Write the following into that file, one line at a time:
#!/bin/sh
curl --head --silent https://missing.csail.mit.edu

The first line might be tricky to get working. It’s helpful to know that # starts a comment in Bash, and ! has a special meaning even within double-quoted (") strings. Bash treats single-quoted strings (') differently: they will do the trick in this case. See the Bash quoting manual page for more information.

[yuzhencode@VM-16-4-centos missing]$ echo '#!/bin/sh' > semester 
[yuzhencode@VM-16-4-centos missing]$ cat semester 
#!/bin/sh
[yuzhencode@VM-16-4-centos missing]$ echo 'curl --head --silent https://missing.csail.mit.edu' >> semester 
[yuzhencode@VM-16-4-centos missing]$ cat semester
