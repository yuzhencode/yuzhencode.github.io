This blog talks about the general solution for k-Sum problem. From 2Sum to 2+1Sum, then 2+1+1Sum. Utimately, the generic solution came out. k-Sum Problem is like Input n integers (positive and negative) and Decide are there k numbers in the input that sum to target integer?

Speaking From 2Sum Problem which is the most basic problem.

1. 2Sum

Given an array of integers nums and an integer target, return indices of the two numbers such that they add up to target. You may assume that each input would have exactly one solution, and you may not use the same element twice. You can return the answer in any order.

Example 1: Input: nums = [2,7,11,15], target = 9 Output: [0,1] Explanation: Because nums[0] + nums[1] == 9, we return [0, 1].

1） Brute Force Solution

It is obvious that two nested loops can exhaust all sums.

def twoSum(self, nums: List[int], target: int) -> List[int]:
    for i in range(0, len(nums)):
        for j in range(i + 1, len(nums)):
            if nums[j] == target - nums[i]:
                return [i, j]

Time complexity: $$O(n^2)$$ Space complexity: $$O(1)$$

2） Use the Hashtable or Hashmap Feature of the Programming Language
def twoSum(self, nums: List[int], target: int) -> List[int]:
    map = {}
    for i, n in enumerate(nums):
        m = target - n
        if m in map:
            return [map[m], i]
        else:
            map[n] = i

Time complexity: $$O(n)$$ Space complexity: $$O(n)$$

3） Two Pointers Technique
def twoSum(self, nums: List[int], target: int) -> List[int]:
    nums_sorted = sorted(nums)
    lo = 0
    hi = len(nums) - 1
    while lo < hi :
        sum = nums_sorted[lo] + nums_sorted[hi]
        if sum < target:
            lo = lo + 1
        elif sum > target:
            hi = hi - 1
        elif sum == target:
            i = nums.index(nums_sorted[lo])
            if nums.count(nums_sorted[hi]) > 1:
                j = nums.index(nums_sorted[hi], i+1)
            else:
                j = nums.index(nums_sorted[hi])
            return i,j

Time complexity: $$O(n)$$ Space complexity: $$O(n)$$ Because this problem needs to return index, extra space is needed to store the sorted array. If the goal is to return numbers, the original space can be used to store them, so that the space complexity is $$O(1)$$.

2. 3Sum

Given an integer array nums, return all the triplets [nums[i], nums[j], nums[k]] such that i != j, i != k, and j != k, and nums[i] + nums[j] + nums[k] == 0. Notice that the solution set must not contain duplicate triplets.

Example 1: Input: nums = [-1,0,1,2,-1,-4] Output: [[-1,-1,2],[-1,0,1]] Explanation: nums[0] + nums[1] + nums[2] = (-1) + 0 + 1 = 0. nums[1] + nums[2] + nums[4] = 0 + 1 + (-1) = 0. nums[0] + nums[3] + nums[4] = (-1) + 2 + (-1) = 0. The distinct triplets are [-1,0,1] and [-1,-1,2]. Notice that the order of the output and the order of the triplets does not matter.

1） Brute Force Solution

Three nested loops would exhaust all sums, which is obviously too complicated, so we drop this approach.

2） Converted to the TwoSum Problem with First Fixed Position
def threeSum(self, nums: List[int]) -> List[List[int]]:
    nums = sorted(nums)
    res = []
    target = 0
    for i in range(0, len(nums)):
        if 0<i<len(nums)-1 and nums[i]==nums[i-1]:
            continue
        tuples=twoSum(nums[i+1:], target-nums[i])
        for tuple in tuples:
            tuple.append(nums[i])
            res.append(tuple)
    return res
def twoSum(nums, target):
    lo = 0
    hi = len(nums)-1
    res = []
    while( lo < hi ):
        left = nums[lo]
        right = nums[hi]
        sum = left + right
        if sum < target:
            while lo<hi and nums[lo]==left:
                lo = lo+1
        elif sum > target:
            while lo<hi and nums[hi]==right:
                hi = hi-1
        else:
            res.append([left, right])
            while lo<hi and nums[lo]==left:
                lo = lo+1
            while lo<hi and nums[hi]==right:
                hi = hi-1
return res

Time complexity: $$O(n^2)$$ Space complexity: $$O(1)$$

4. k-Sum

Problem: k-Sum
Input:n integers (positive or negative)
Decide: Are there k (distinct) numbers in the input which sum to Target integer?

The number of K can be any number greater than 2, as long as it does not exceed the size of the array.

Constraints：

  • Whether the array is ordered or not？
  • Whether the elements in the array are distinct？
  • Whether the sub array is uniquely determined？

We use the generic approach, assuming that the array is unordered, that the array elements are duplicated and that the resulting subarray is not uniquely determined and may be 0 or more.

A generalized solution to the kSum problem using the idea of recursion.
def nSum(nums, n, target):
    nums = sorted(nums)
    res = []
    if n < 2 or len(nums) < 2:
        print("")
    elif n==2:
        lo = 0
        hi = len(nums)-1
        res = []
        while( lo < hi ):
            left = nums[lo]
            right = nums[hi]
            sum = left + right
            if sum < target:
                while lo<hi and nums[lo]==left:
                    lo = lo+1
            elif sum > target:
                while lo<hi and nums[hi]==right:
                    hi = hi-1
            else:
                res.append([left, right])
                while lo<hi and nums[lo]==left:
                    lo = lo+1
                while lo<hi and nums[hi]==right:
                    hi = hi-1
        return res  
    else:      
        for i in range(0, len(nums)):
            if 0<i<len(nums)-1 and nums[i]==nums[i-1]:
                continue
            tuples=nSum(nums[i+1:], n-1, target-nums[i])
            if tuples != None:
                for tuple in tuples:
                    tuple.append(nums[i])
                    res.append(tuple)
        return res

Time complexity: $$O(n^{k-1})$$ Space complexity: $$O(1)$$
